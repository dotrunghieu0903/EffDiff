1. VẤN ĐỀ: Tại sao Image Reward > 1.0?

   ✓ KHÔNG PHẢI LỖI - Đây là hành động bình thường!
   
   ImageReward model sử dụng:
   - Phân phối chuẩn N(0, 1) nhưng KHÔNG bounded [-1, 1]
   - Công thức: score = (raw_score - 0.167) / 1.033
   
   Với FLICKR30K dataset:
   - 40% scores > 1.0 (vs expected 16% cho N(0,1))
   - Mean score: 0.776 (có thể lên đến 1.0+ với nhiều ảnh)
   - Max score: 1.806
   - Min score: -0.949
   
   ➜ Captions FLICKR30K có chất lượng TỐNHẤT so với COCO
     (captions cụ thể, mô tả chi tiết)

2. SO SÁNH DATASETS:

   FLICKR30K (captions thủ công, chất lượng cao):
   - Mean: 0.77 - 1.05
   - Scores > 1.0: 40-60%
   - Mô tả: Captions cụ thể, phù hợp tốt với image

   COCO (captions crowdsourced, generic):
   - Mean: 0.3 - 0.5
   - Scores > 1.0: 10-20%
   - Mô tả: Captions ngắn, generic


3. GIẢI PHÁP ĐỀ XUẤT:

   Nếu muốn scores trong [0, 1]:
   
   Option A: TANH (Smooth, Recommended)
   ───────────────────────────────────────
   score_normalized = (tanh(score) + 1) / 2
   
   Kết quả:
   - Range: [0.13, 0.97]
   - Mean: 0.752
   - Preserves relative ordering
   
   Example:
   >>> from shared.metrics import compute_image_reward
   >>> avg_score = compute_image_reward(
   ...     'output_dir', 
   ...     captions_dict,
   ...     normalize_method='tanh'
   ... )
   
   
   Option B: SIGMOID (Probabilistic)
   ──────────────────────────────────
   score_normalized = 1 / (1 + exp(-score))
   
   Kết quả:
   - Range: [0.28, 0.86]
   - Mean: 0.668
   - Interpretation: likelihood score
   
   
   Option C: CLAMP (Preserve distances)
   ────────────────────────────────────
   score_normalized = (clamp(score, -2, 2) + 2) / 4
   
   Kết quả:
   - Range: [0.26, 0.95]
   - Mean: 0.694
   - Best for ranking


4. SỬ DỤNG TRONG CODE:

   # ❌ Không normalize (original scores)

   # ✅ Normalize để có [0, 1]
